# CarQuest 游戏设定圣典

版本：0.1  
目标硬件：M5Stack Cardputer ADV + Cap LoRa/GPS  
设计目标：一个由真实移动驱动的小型奇幻冒险游戏，感觉像浓缩版桌面 RPG，同时可以作为车载终端上的自动冒险日志运行。

## 1. 核心概念

CarQuest 是一个基于位置和移动状态的公路奇幻 RPG。玩家是一名流浪冒险者，车辆被视为一辆带有魔法性质的远征车。现实世界中的移动会转化为探索；距离、速度、GPS 状态和 6 轴运动数据会生成遭遇、宝箱、危险和营地时刻。

玩家在移动过程中不需要操作游戏。行驶时，CarQuest 会自动运行成一份冒险日志；车辆停止后，游戏进入营地模式，玩家可以治疗、开箱、整理战利品、制作升级，并为下一段旅途做准备。

整体气质是“龙与地下城遇上车载仪表伙伴”：怪物、遗物、武器、补给、路边营地、被诅咒的信号和奇怪的路线，先通过小屏文字表达，之后再通过紧凑图标强化。

## 2. 核心循环

1. 旅行
   - GPS 距离和 IMU 运动数据累积探索进度。
   - 每隔固定距离，游戏掷出一次事件。
   - 强烈运动、急加速、刹车或剧烈转向会提高危险和奖励。

2. 遭遇
   - 事件可能是怪物、危险、宝藏、神龛、商人、道路预兆或 LoRa 信号。
   - 移动中的遭遇根据玩家属性、装备、运动加成和随机掷骰自动结算。

3. 收益或代价
   - 胜利获得 XP、金币、废料、钥匙、药水、遗物碎片或装备。
   - 失败损失 HP、金币、补给，或被迫进入紧急营地。

4. 营地
   - 车辆停止一段时间后触发。
   - 玩家可以休息、使用药水、开箱、制作装备、查看背包和回顾日志。

5. 成长
   - XP 提升等级。
   - 等级提升 HP 和基础属性。
   - 装备和遗物形成不同构筑。
   - 更高等级解锁更稀有的怪物、更好的宝藏和更深的路线事件。

## 3. 玩家角色

玩家由一组适合 204x109 小屏显示的紧凑属性表示。

### 主要属性

| 属性 | 缩写 | 含义 | 默认 |
|---|---:|---|---:|
| 等级 | L | 角色整体层级 | 1 |
| 生命值 | HP | 承受伤害的能力 | 30 |
| 攻击 | ATK | 进攻能力 | 6 |
| 防御 | DEF | 减伤与生存能力 | 3 |
| 专注 | FOC | 对抗魔法、恐惧、信号和陷阱事件 | 3 |
| 幸运 | LUK | 提升宝物、暴击、稀有事件和宝箱品质 | 2 |
| 经验 | XP | 等级进度 | 0 |
| 金币 | G | 货币 | 0 |

当前固件已经包含等级、HP、ATK、DEF、XP、金币。Focus 和 Luck 适合在 UI 有空间或有详情页后加入。

### 派生属性

| 派生属性 | 公式 | 说明 |
|---|---|---|
| 最大 HP | `24 + Level * 6 + END bonus` | 当前固件每级 +6 HP，适配这个方向。 |
| 战力 | `Level * 6 + ATK + weapon bonus` | 自动战斗使用。 |
| 防护 | `DEF + armor bonus + shield bonus` | 降低受到的伤害。 |
| 意志 | `FOC + relic bonus` | 对抗诅咒、信号幽灵、恐惧和幻象。 |
| 掉落评级 | `LUK + route tier + chest tier` | 决定宝物品质。 |
| 驾驶加成 | `clamp(MotionScore * 2, 0, 8)` | 现有运动加成可以继续保留。 |

### 角色原型

原型是可选的轻量构筑。第一版可以没有职业，之后在 3 级解锁。

| 原型 | 奇幻定位 | 公路奇幻风味 | 属性倾向 |
|---|---|---|---|
| 公路骑士 | 战士 | 重型车辆守护者 | +HP、+DEF |
| 信号法师 | 法师 | 使用 GPS/LoRa 预兆 | +FOC、遗物效果 |
| 废料游侠 | 游侠 | 拾荒者与路线斥候 | +ATK、+LUK |
| 齿轮牧师 | 牧师 | 修复与守护 | +治疗、+营地行动 |
| 漂移盗贼 | 盗贼 | 风险与收益专家 | +LUK、暴击率 |

## 4. 等级成长

CarQuest 应使用适合短局体验的 XP 数值。玩家应在几次有意义的出行后升级，而不是长时间刷经验。

### XP 曲线

推荐公式：

```text
升到下一级所需 XP = 16 + Level * 18 + floor(Level^2 * 2)
```

更简单的固件公式：

```text
所需 XP = Level * 18
```

早期原型可以使用 `Level * 18`。当内容深度增加后，应替换为扩展曲线。

### 等级表

| 等级 | 升级所需 XP | 最大 HP | ATK | DEF | 解锁 |
|---:|---:|---:|---:|---:|---|
| 1 | 18 | 30 | 6 | 3 | 基础旅行事件 |
| 2 | 36 | 36 | 7 | 4 | 1 阶装备制作 |
| 3 | 60 | 42 | 8 | 4 | 选择原型 |
| 4 | 84 | 48 | 9 | 5 | 遗物槽 1 |
| 5 | 118 | 54 | 10 | 5 | 精英怪物 |
| 6 | 156 | 60 | 11 | 6 | 营地升级 |
| 7 | 202 | 66 | 12 | 6 | 遗物槽 2 |
| 8 | 256 | 72 | 13 | 7 | 路线首领 |
| 9 | 318 | 78 | 14 | 7 | 传奇战利品 |
| 10 | 390 | 84 | 15 | 8 | 终局路线预兆 |

### 升级奖励

每级：

- 最大 HP +6
- HP 回满
- ATK +1

每个偶数等级：

- DEF +1

每三个等级：

- 选择一个恩惠：
  - Focus +1
  - Luck +1
  - 最大药水容量 +1
  - 宝箱品质 +1
  - 营地治疗 +1

## 5. 属性与掷骰系统

CarQuest 应该有桌面 RPG 的味道，但公式必须足够小，方便嵌入式实现。

### 核心检定

```text
分数 = 属性 + 装备 + 等级加成 + 驾驶加成 + random(0..10)
```

推荐结果区间：

| 差值 | 结果 |
|---:|---|
| `Score >= Difficulty + 8` | 大成功 |
| `Score >= Difficulty` | 成功 |
| `Score >= Difficulty - 5` | 部分成功 |
| 其他 | 失败 |

### 战斗检定

```text
PlayerPower = Level * 6 + ATK + WeaponPower + DriveBonus + random(0..10)
MonsterPower = MonsterLevel * 7 + Danger + MonsterTrait + random(0..8)
```

如果 `PlayerPower >= MonsterPower`，玩家获胜。

如果玩家失败：

```text
Damage = max(1, MonsterPower - PlayerPower / 2 - Guard / 2)
```

### 暴击

大成功触发条件：

```text
PlayerPower >= MonsterPower + 8
```

暴击奖励：

- XP +25%
- 金币 +25%
- 更高几率获得废料、钥匙或遗物碎片

大失败触发条件：

```text
MonsterPower >= PlayerPower + 10
```

大失败效果：

- 额外 HP 损失
- 丢失一小包金币
- 如果实现装备耐久，则损失耐久

## 6. 移动与传感器规则

CarQuest 应把真实移动转化为奇幻压力。

### 旅行状态

| 信号 | 游戏含义 |
|---|---|
| GPS 距离 | 探索进度 |
| GPS 速度 | 路线节奏 |
| IMU 运动分数 | 道路危险与战斗动量 |
| 停止时长 | 营地触发 |
| 卫星数量 | 导航确定性 |
| LoRa 消息 | 附近冒险者或队伍信号 |

### 路线危险

```text
RouteDanger = clamp(MotionScore * 10 + random(0..40), 0, 100)
```

| 危险值 | 事件倾向 |
|---:|---|
| 0-24 | 安全旅行、小宝藏、商人 |
| 25-44 | 宝箱、小危险、弱怪 |
| 45-69 | 标准怪物、陷阱、更好宝物 |
| 70-89 | 精英怪、诅咒事件、稀有宝箱 |
| 90-100 | 首领预兆、伏击、传奇机会 |

### 事件距离

推荐：

```text
下一事件距离 = 当前距离 + random(160m..360m)
```

之后路线类型可以调整该距离：

| 路线类型 | 事件间隔 | 风味 |
|---|---:|---|
| 城市 | 120-240m | 更多小事件 |
| 高速 | 300-700m | 更少但更强事件 |
| 山路 | 180-420m | 更多危险 |
| 未知道路 | 140-500m | 更多怪异事件 |

## 7. 战斗系统

战斗必须在移动过程中自动结算。营地可以显示总结并允许准备，但不应要求行驶时实时操作。

### 战斗阶段

1. 遭遇出现。
2. 游戏计算路线危险和怪物等级。
3. 玩家战力由等级、ATK、武器、遗物和驾驶加成计算。
4. 怪物战力由怪物等级、家族、危险和特性计算。
5. 结算结果。
6. 用一行短日志记录结果。

### 伤害类型

| 类型 | 用途 |
|---|---|
| 物理 | 野兽、掠夺者、护甲检定 |
| 奥术 | 灵体、诅咒、遗物事件 |
| 信号 | GPS/LoRa 幽灵、静电实体 |
| 道路 | 危险、陷阱、刹车、剧烈运动 |

小型固件版本可以只保留一个 HP 和一个 DEF。伤害类型主要用于命名和未来抗性。

### 状态效果

| 状态 | 效果 | 持续 |
|---|---|---|
| 受伤 | ATK -1 | 直到营地休息 |
| 震慑 | FOC -1 | 接下来 2 个事件 |
| 干扰 | LoRa/GPS 事件惩罚 | 下一个事件 |
| 鼓舞 | 下个事件战力 +2 | 下一个事件 |
| 防护 | 受到伤害 -2 | 下一个事件 |
| 幸运 | 下次掉落 +1 | 下次宝物 |

## 9. 怪物缩放

推荐怪物等级：

```text
MonsterLevel = clamp(PlayerLevel + random(-1..2), 1, 10)
```

危险修正：

```text
Danger = FamilyBaseDanger + RouteDangerBandBonus
```

路线危险区间加成：

| 路线危险 | 加成 |
|---:|---:|
| 0-24 | -1 |
| 25-44 | 0 |
| 45-69 | +2 |
| 70-89 | +4 |
| 90-100 | +7 |

### 怪物奖励

```text
XP = 4 + MonsterLevel * 3 + Danger
Gold = random(1..4 + MonsterLevel * 2)
```

精英倍率：

- XP x1.25
- 金币 x1.25
- 稀有掉落 +10%

首领倍率：

- XP x2
- 金币 x2
- 保底遗物碎片或独特物品

## 13. 营地系统

营地是主要交互模式。它应该安静、有用、适合整理。

### 营地触发

```text
停止 15 秒后，进入营地。
```

### 营地动作

| 动作 | 效果 | 条件 |
|---|---|---|
| 休息 | 少量治疗 | 营地模式 |
| 使用药水 | 大量治疗 | 药水 > 0 |
| 打开宝箱 | 消耗钥匙，获得战利品 | 钥匙 > 0 |
| 升级武器 | 消耗废料，+ATK | 废料足够 |
| 修理装备 | 移除受伤/干扰 | 废料或修理包 |
| 查看日志 | 查看近期事件 | 总是可用 |
| 管理遗物 | 装备遗物 | 已解锁遗物 |
| 队伍信号 | 查看附近 LoRa 玩家 | LoRa 可用 |

### 休息规则

原型：

```text
休息治疗 4 HP。
```

扩展：

```text
休息治疗 4 + Level + 营地加成。
```

可选长休：

```text
停止 10 分钟后，每个现实会话可完全治疗一次。
```

## 14. 事件系统

事件应该短、可读、快速结算。

### 事件类别

| 类别 | 示例 | 结果 |
|---|---|---|
| 怪物 | 道路幽魂袭击 | 战斗 |
| 宝物 | 发现隐藏缓存 | 掉落检定 |
| 危险 | 被诅咒的坑洞 | 道路伤害或废料 |
| 神龛 | 里程标记发光 | 治疗或增益 |
| 商人 | 流浪机械师 | 买卖/制作 |
| 信号 | LoRa/GPS 异常 | Focus 检定 |
| 任务 | 跟随预兆一段距离 | 长期奖励 |
| 首领预兆 | 龙影掠过道路 | 多阶段事件 |

### 事件选择

```text
Risk = MotionScore * 10 + random(0..40)
```

| Risk | 事件 |
|---:|---|
| 0-25 | 宝物、神龛、商人 |
| 26-45 | 宝物、小危险 |
| 46-70 | 标准怪物 |
| 71-88 | 精英怪或诅咒 |
| 89-100 | 首领预兆或稀有宝物 |

### 路线任务示例

| 任务 | 触发 | 目标 | 奖励 |
|---|---|---|---|
| 三里诅咒 | 击败 3 个幽魂 | 3 个事件内不被 KO | 遗物碎片 |
| 锈蚀合唱 | 找到 5 个废料 | 在营地升级 | 废料护符 |
| 龙之地下道 | 高危险首领预兆 | 存活首领事件 | 龙钥匙 |
| 失落坐标 | GPS 漂移事件 | 行驶 1km | 地图碎片 |

## 15. LoRa 多人想法

LoRa 应该像附近冒险者擦肩而过的信号。

### 信标包

原型格式：

```text
CQ1|id|level|distance|mode
```

### 多人效果

| 附近信号 | 效果 |
|---|---|
| 同等级附近 | 下一事件士气 +1 |
| 高等级附近 | 小概率获得援助 |
| 低等级附近 | 玩家获得守护加成 |
| 多个信号 | 队伍光环，掉落机会提高 |
| 求救信号 | 可选救援事件 |

### 队伍光环

```text
PartyBonus = min(3, nearbyPlayers)
```

效果：

- 战斗检定 +PartyBonus
- 宝物检定 +PartyBonus
- 稀有共享首领预兆机会

## 22. 设计原则

- 永远不要要求驾驶中操作。
- 让移动有魔法感，但不能分散注意力。
- 每个事件都应一眼可读。
- 把营地作为做决策的地方。
- 优先使用可靠的小数字，而不是复杂隐藏系统。
- 通过掉落表、怪物家族和遗物增加深度，而不是膨胀 UI。
- 把 GPS、IMU 和 LoRa 当作奇幻输入：距离、危险、预兆、队伍。
